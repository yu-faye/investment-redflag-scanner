Investment Red-Flag Scanner

Deterministic offensive-audit detectors against Middelborg-linked Nordic issuers, with click-through provenance to the originating PDF page.

Pipeline

Five deterministic steps from issuer PDF to traceable finding. Every step's output is auditable on disk.

Step 1
Ingest
Pull issuer PDFs, extract text + curated metrics, hash + version-pin every byte in sources.json.
Step 2
Hypothesise
Each audit claim is a falsifiable JSON object in hypotheses.json with explicit questions and peer controls.
Step 3
Gather evidence
Independent taps (public-procurement registries, company registry, derived cross-tap analyzers) emit standardised verdicts. Every raw response is cached with sha256.
Step 4
Triangulate
A single rule-bound engine combines verdicts. Severity is derived, never set by any one tap. Single-source absence cannot drive critical.
Step 5
Trace
Every finding has a Sankey lane, an auditor narrative paragraph, and a nested argument tree with citations linking back to the cache JSON or the PDF page.

Logic chain landscape

Every finding flows left-to-right: issuer → claim/rule → falsification question → tap evidence (verdict) → derived severity. Click any node to filter the leaderboard above to only those findings whose reasoning passes through that node. Lane width = number of distinct findings using that path.

Triangulation matrix

Every audit hypothesis (rows) cross-checked against every available external data source family (columns). Cell verdicts: ✓ confirms, ◐ partial, ✗ refutes, ø not_found, ◦ neutral, ! error, — not yet queried. A claim is only allowed to graduate to critical when at least two independent taps agree AND every blocking_for_critical falsification question has been addressed.

Audit roadmap

Which external data source would maximally raise triangulation coverage across all open hypotheses next. Ranked by total expected information gain. Implementing one of these adds a new column to the matrix above.
