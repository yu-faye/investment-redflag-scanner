Investment Red Flag Scanner

Deterministic offensive audit detectors against Nordic issuers tied to Middelborg. Every finding traces back to the exact page of the original report.

How it works

Five simple steps. The whole pipeline is reproducible and auditable, no opaque models in the loop today.

Step 1
Collect
Pull published financial reports from issuers and version pin every byte.
Step 2
Semantic rules
Apply deterministic semantic rules to spot claims worth checking. Coming soon: Claude API for deeper language analysis on disputed claims.
Step 3
Evidence gathering
Cross check each claim against independent public registries (procurement, company registry, EU tenders).
Step 4
Triangulate
Combine evidence streams into one verdict per claim. No single source can drive a critical flag on its own.
Step 5
Trace
Every finding links back to the exact source page so any reader can re derive the conclusion.

Logic chain landscape

Every finding flows left to right: issuer, claim or rule, falsification question, tap evidence (verdict), derived severity. Click any node to filter the leaderboard above to only the findings whose reasoning passes through that node. Lane width is the count of distinct findings using that path.

Triangulation matrix

Every audit hypothesis (rows) cross checked against every available external data source family (columns). Cell verdicts: ✓ confirms, ◐ partial, ✗ refutes, ø not_found, ◦ neutral, ! error, — not yet queried. A claim only graduates to critical when at least two independent taps agree AND every blocking_for_critical falsification question has been addressed.

Audit roadmap

Which external data source would maximally raise triangulation coverage across all open hypotheses next. Ranked by total expected information gain. Implementing one of these adds a new column to the matrix above.
